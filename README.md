## 📚 学术研究基础

本产品规划基于以下四篇最新学术论文：

### 1. LLM4PLC (ICSE-SEIP 2024)
**文件**: `2401.05443v1.pdf`
- **核心贡献**: 引入用户反馈+语法检查+编译器+SMV模型检验的完整验证流水线
- **关键数据**: 生成成功率从 47% 提升到 72%，专家评分从 2.25 提升到 7.75/10
- **技术路线**: LoRA微调 + 多级验证策略

### 2. Agents4PLC
**文件**: `2410.14209v2.pdf` (arXiv:2410.14209)
- **核心贡献**: 首个多智能体闭环系统
- **工作流**: 需求→规划→编码→编译→形式验证→调试
- **创新点**: 发布首个自然语言到ST代码的基准测试集

### 3. AutoPLC
**文件**: `2412.02410v2.pdf` (arXiv:2412.02410)
- **核心贡献**: 四阶段流水线（厂商API库 + 案例库 + IDE动态验证）
- **关键数据**: 914个任务上实现 90%+ 编译通过率，单任务成本 $0.13
- **验证平台**: Siemens TIA Portal + CODESYS

### 4. IEC 图形语言探索
**文件**: `2410.15200v1.pdf` (arXiv:2410.15200)
- **核心贡献**: 探索 Ladder/SFC 图形化编程语言的 AI 生成可行性
- **关键发现**: GPT-4 可生成可读 SFC，Ladder 需要中间表示+渲染方案
- **技术路径**: JSON/文本中间表示 → 库化常见梯级 → RAG+图语法限制

## 💡 核心价值主张

### 技术可行性 ✅
- 三篇顶会/顶刊论文验证了技术路径
- 编译通过率达 90%+
- 推理成本仅 $0.13/任务
- 已有成熟的验证流水线（编译+形式验证+仿真）

### 市场机会 📈
- Siemens、Rockwell 等厂商 Copilot 仅支持自家生态
- 现有方案几乎只支持 ST，缺少 Ladder/SFC 支持
- 12-18个月时间窗口期，跨品牌方案仍然空白
- 客户主动催进度，市场需求强烈

### 差异化优势 🎯
1. **跨品牌支持**: Siemens + CODESYS + Rockwell + Beckhoff
2. **多语言覆盖**: ST + Ladder + SFC 全栈
3. **私有化部署**: 支持国内算力与完全离线环境
4. **成本优势**: 可插拔推理源，无需绑定 Azure OpenAI
